## Blockers (pre-implementation)

### A1 — `evaluate` 파라미터 재설계
현행: `evaluate(context, rulePack, registry, lockedSlots)` — 위치 기반 4-인수.
변경: `evaluate(context, { rules, registry, lockedSlots, policy?, trace? })` — options bag.
이유: Premise 5의 locked slot이 1차 primitive이며, Phase 1b 확장(policy · risk · trace) 시 파괴 변경 회피.

### A2 — Trace 출력 포맷 실체 선고정
현행: "The Assignment"에서 10–20분 후속 작업으로 유보.
변경: `docs/TRACE-FORMAT-v1.txt`를 구현 착수 전 선행 산출물로 격상.
이유: trace가 제품 whoa의 유일한 비가격 차별화. 픽스처로 고정되지 않으면 `formatTrace` 설계가 코드 중 흔들림 · byte-determinism 테스트 기준 부재.

### A4 — `lockedSlots` 인터페이스 형태 고정
권고 구조:
```ts
interface LockedSlot {
  readonly slot: string;
  readonly componentKey: string;
  readonly reason: 'business-fixed' | 'ad' | 'compliance' | 'promotion';
  readonly priority?: number;
}
type LockedSlots = readonly LockedSlot[];
```
이유: 배열 기반이 trace 표기 · JSON 직렬화 · 결정론적 정렬에 최적. 공개 후 변경 시 Premise 5 검증 기준(3개월 내 사용 사례) 충족 불가.

### T1 — Byte-determinism 테스트 필수
`fixtures/trace-basic.txt` 대조. 같은 입력 → 동일 문자열 trace. 회귀 방지의 1차 보호선.

## Code Quality Constraints (사전 규약)

사용자 글로벌 CLAUDE.md 충돌:
- 200 LoC 파일 상한 vs 설계 150–250 LoC 목표.
- 해결: 4-파일 분할.
  - `src/evaluate.ts` (≤150)
  - `src/trace.ts` (≤80)
  - `src/types.ts` (≤80)
  - `src/index.ts` (re-export)
- 50 LoC 함수 상한: `evaluate` 본체 5단계를 내부 단계 함수로 분해.

필수 준수:
- `strict: true` · `any` 금지 · 모든 return 타입 명시.
- `interface` for object shapes · `type` for unions · `readonly` 기본값.
- Named export only.
- Discriminated union for `TraceEntry`.
- Pure function — side-effect free · runtime regex compile 금지.
