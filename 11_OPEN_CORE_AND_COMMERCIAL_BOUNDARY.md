# 11. Open Core and Commercial Boundary

## 1. Open-source core
Should include:
- schema
- evaluator
- orchestrator core
- registry contracts
- trace engine
- simulator
- sample packs
- examples

## 2. Commercial layers
Should include:
- hosted control plane
- remote evaluation service
- approval workflows
- audit logs
- enterprise policy dashboard
- premium vertical packs
- optimization analytics

## 3. Why this boundary works
Developers adopt open standards.
Enterprises pay for governance, hosted operations, and managed safety.

## 4. Risks
1. open core too thin
2. commercial layer too vague
3. pack model confusion
4. hosted dependency fear

## 5. Mitigation
- strong local runtime
- excellent docs
- clear API contracts
- bring-your-own-provider support
- transparent hosted value

## 6. Strategic note
Commercial value should come from control plane, governance, and optimization,
not from crippling the core runtime.
